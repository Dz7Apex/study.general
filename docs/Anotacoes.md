## Terminal
- comando para rodar o site: `npm run dev`
- comando para interromper execução no terminal: ctrl + c
- comando para limpar o terminal: ctrl + L


## Caixas
- comando para verificar a situação do stage do git: git status
- comando para adicionar itens para o stage: `git add localização_arquivo` | exemplo: `git add ../../../docs/Anotacoes.md` | exemplo para adicionar tudo no stage: `git add .`
- comando para duplicar linha: shift + alt + cima ou baixo
- comando para adicionar itens do stage em um commit: `git commit -m "mensagem do que foi feito"`
  - `-m`: representa o que o commit está fazendo. é equivalente a escrever com pincel atomico em uma caixa, informando se a caixa é de roupas, remedios ou eletronicos
- comando para enviar os commits para o servidor (sincronizar): `git push`


## Pastas
- comando para entrar em uma pasta: `cd pasta` 
- comando para voltar uma pasta: `cd ..`
- comando para voltar duas pasta: `cd ../..`
- comando para voltar tres pasta: `cd ../../..`

### Observações

> importante:
> - repetir e exercitar 
> - stage do git: são como livros selecionados
> - commit do git: são como livros que foram empacotados dentro de uma caixa, mas, essa e outras caixas (commits) ainda estão no mesmo local que foram embalados. para que as caixas cheguem ao destinatario é necessário executar o comando `git push`
> - push do git: representa a ação de enviar todas as caixas (commits) para um destino
> - cd: significa Change Directory (mudar de diretório)
> - cada mudança vira um commit
